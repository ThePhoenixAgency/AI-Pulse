Last updated: February 16, 2026

What Are OpenClaw Skills?

OpenClaw skills are folders containing instructions, scripts, and resources that teach AI agents how to perform specific tasks. They follow the AgentSkills open standard, making them compatible with multiple AI agent platforms including Claude Code, Cursor, Windsurf, and more.

Each skill contains a SKILL.md file with YAML frontmatter defining the skill's name, description, and requirements. The body contains instructions the agent reads to learn the capability.

The 5 Main Sources for OpenClaw Skills

1. ClawHub — The Official Marketplace

URL: clawhub.com

ClawHub is OpenClaw's official skill registry. It offers:

  • Curated, verified skills
  • Version management
  • One-command installation
  • Automatic updates

How to install from ClawHub:

# Install the ClawHub CLI
npm i -g clawhub

# Search for skills
clawhub search "postgres backups"

# Install a skill
clawhub install my-skill

# Install specific version
clawhub install my-skill --version 1.2.3

# Update all installed skills
clawhub update --all

4. GitHub Repositories — Direct Install

Any GitHub repository with a valid SKILL.md can be installed directly into OpenClaw.

How it works:

  1. Find a repo with skills (look for SKILL.md files)
  2. Give OpenClaw the URL
  3. OpenClaw parses the repo and lets you choose which skill to install

Example repositories with skills:

  • anthropics/skills — Official Anthropic skills (PDF, DOCX, XLSX, webapp-testing)
  • vercel-labs/agent-skills — React, Next.js, web design patterns
  • obra/superpowers — Systematic debugging, brainstorming, TDD
  • supabase/agent-skills — Postgres best practices
  • expo/skills — React Native development

For AI agents: When you encounter a GitHub repo mentioned as having skills, you can install directly by providing the URL. Check the repo's README for available skills.

5. Bundled Skills — Ships with OpenClaw

OpenClaw ships with built-in skills for common operations. These require no installation.

Skill locations (by precedence):

  1. Workspace skills: <workspace>/skills (highest priority)
  2. Managed/local skills: ~/.openclaw/skills
  3. Bundled skills: Ships with npm package (lowest priority)

Common bundled skills include:

  • Weather lookups
  • Web search and fetch
  • Browser automation
  • File operations
  • System commands

For AI agents: Bundled skills are always available. Check <available_skills> in your system context to see what's loaded. You don't need to install these.

Security Best Practices

The OpenClaw community has learned hard lessons about skill security:

Before Installing Any Skill:

  1. Verify the publisher — Prefer established publishers with track records
  2. Audit the code — Read SKILL.md and any scripts before enabling
  3. Check permissions — Understand what the skill needs access to
  4. Use sandboxing — Run untrusted skills in sandboxed mode
  5. Monitor activity — Watch for unexpected network calls or file access

For Crypto/Financial Skills:

  • Never connect to your main wallet
  • Use dedicated wallets with limited funds
  • Set up independent transaction monitoring
  • Grant only required permissions
  • Regular audit installed skills for updates

Known Security Incidents:

  • CVE-2026-1847 (DataBridge) — Sandbox escape via deserialization vulnerability. Patched within 36 hours.
  • BankrBot malicious packages — 386 malicious skills identified by Snyk in early 2026.

FAQ

What is the difference between ClawHub and skills.sh?

ClawHub is OpenClaw's official marketplace with curated skills and version management. skills.sh is a broader directory that indexes skills from GitHub repos and works with multiple AI agents, not just OpenClaw.

Can I use skills from other AI agents?

Yes! OpenClaw follows the AgentSkills open standard. Skills created for Claude Code, Cursor, Windsurf, and other compatible agents work with OpenClaw.

How do I know if a skill is safe?

  • Check the publisher's reputation
  • Read the code before installing
  • Look for verification badges on ClawHub
  • Search for security advisories
  • Start with sandboxed execution for untrusted skills

Where are installed skills stored?

By default: ~/.openclaw/skills/ for managed skills, or <workspace>/skills/ for workspace-specific skills.

How do I uninstall a skill?

Delete the skill folder from your skills directory, or use clawhub remove skill-name if installed via ClawHub.
