Open Slop

Because your repository isn't a training ground for prompt engineers.

Open Slop is a forensic triage bot that protects maintainers from the wave of "drive-by" AI-generated Pull Requests.

We don't use AI to fight AI. We don't scan code for "AI fingerprints" (which doesn't work anyway). Instead, we look at the one thing a spammer can't fake: Time and Behavior.

When a PR is opened, Open Slop instantly acts as a BS-detector and leaves a single triage comment for the maintainer based on three ruthless metrics:

  • The Velocity Signal: Did they fork the repo, "understand" the architecture, and write 200 lines of complex logic in 42 seconds? Flagged.

  • The Shotgun Signal: Did this user open 15 PRs in 15 completely unrelated repositories while you were having breakfast? Flagged.

  • The Ghost Signal: Is the account older than the milk in your fridge, or was it created yesterday? Flagged.

Stop manually reviewing hallucinations. Let Open Slop separate the real contributors from the resume-farmers.

Usage

Add this to your repository's 

.github/workflows/open-slop.yml
:

name: Open Slop on: pull_request: types: [opened] permissions: pull-requests: write contents: read jobs: triage: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: dr-alberto/open_slop@main with: github-token: ${{ secrets.GITHUB_TOKEN }}

Development

npm install
npm run build # bundles src/ into dist/ via @vercel/ncc

Important: Commit 

dist/
after building — GitHub Actions runs directly from the repo without installing dependencies.
