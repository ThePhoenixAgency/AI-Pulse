Qu'est ce que sed ?

La commande sed est un éditeur de flux utilisé pour filtrer et transformer du texte.

Un éditeur de flux est utilisé pour effectuer des transformations de texte basiques sur un flux d'entrée (un fichier ou l'entrée d'un pipe) contrairement à un éditeur de texte classique qui ne gère qu'un fichier.
Mais c'est sa capacité à filtrer du texte dans un tube qui le distingue des autres éditeurs.

Cette commande fait partie du paquet sed qui est disponible et installé dans toutes les distributions Linux.

La syntaxe de la commande est la suivante :

Code BASH :

sed [options] 'commande' [fichier...]

- commande : la commande qu'on va faire exécuter à sed
- fichier... : le(s) fichier(s) dans le(s)quel(s) on souhaite effectuer la recherche.

La commande sed lit l'entrée standard si aucun fichier n'est spécifié, elle s'utilisera donc parfois sous cette forme :

Code BASH :

commande | sed [options] 'commande'

Quelques options que j'utilise souvent :
-i : Modifie directement le fichier original (attention, irréversible sans backup).
-i.bak : Modifie le fichier et crée une sauvegarde .bak avant modification.
-e : Permet d'enchaîner plusieurs commandes sed (ex: -e '...' -e '...').
-E : Permet de traiter des expressions régulières complexes.

Dans la suite de ce mémo, je n'utilise pas l'option -i pour ne pas remplacer le fichier et que les résultats soient affichés sur le terminal.
Ainsi, vous pouvez tester sans altérer le fichier original.

Je vous propose ce petit fichier d'exemple pour tester :

Code BASH :

cat <<EOF > /tmp/sed.txt
# Fichier de configuration de test
USER=Adrien
PORT=8080
DEBUG=yes
Ce fichier contient du texte pour pratiquer.
Il y a des mots comme salut ici.
Et un autre salut là et un autre salut ici.
Start
Données confidentielles
End
# Chemin
/var/log/httpd
# Fin du fichier
EOF

Utiliser sed

Les commandes se placent entre les guillemets simples '...'.

La substitution

C'est la fonction la plus utilisée pour remplacer du texte. Elle se définit avec "s". Voici quelques exemples.

Remplacer la première occurrence par ligne :

Code BASH :

sed 's/salut/bonjour/' /tmp/sed.txt

Une ligne contenant salut a sa première occurence modifiée et chaque ligne est traitée :

Code TEXT :

Il y a des mots comme bonjour ici.
Et un autre bonjour là et un autre salut ici.

Remplacer toutes les occurrences d'une ligne (Global) :

Code BASH :

sed 's/salut/bonjour/g' /tmp/sed.txt

Une ligne contenant salut a toutes ses occurences modifiées :

Code TEXT :

Et un autre bonjour là et un autre bonjour ici.

On peut évidemment changer plein de choses quelque soit la ligne :

Code BASH :

sed 's/PORT=8080/PORT=443/' /tmp/sed.txt

Si on veut ignorer la casse (par défaut sed est sensible à la casse) :

Code BASH :

sed 's/adrien/brooke/I' /tmp/sed.txt

La suppression

La suppression se définit avec "d". Voici quelques exemples.

Supprimer la ligne numéro 2 :

Code BASH :

sed '2d' /tmp/sed.txt

On n'a pas encore parlé d'expression régulières, mais voici un exemple. Supprimer les lignes commençant par un # :

Code BASH :

sed '/^#/d' /tmp/sed.txt

Supprimer les lignes vides :

Code BASH :

sed '/^$/d' /tmp/sed.txt

Et si on veut combiner les 2 :

Code BASH :

sed -e '/^$/d' -e '/^#/d' /tmp/sed.txt

On peut le faire avec des regexp avancées mais oure cela on a besoin de l'option -E :

Code BASH :

sed -E '/^($|#)/d' /tmp/sed.txt

A l'inverse de la suppression, on peut vouloir extraire des infos.
Cela se définit avec "p" combiné avec l'option -n. Voici quelques exemples.

Afficher uniquement la ligne 3 :

Code BASH :

sed -n '3p' /tmp/sed.txt

On pourra extraire des infos entre deux autres comme ceci :

Code BASH :

sed -n '/Start/,/End/p' /tmp/sed.txt

Utilisation avancée de sed

On pourra rechercher et remplacer un motif.
Ici, voici un exemple pour chercher une ligne commençant par DEBUG et la commentant :

Code BASH :

sed '/^DEBUG/s/^/# /' /tmp/sed.txt

Parfois, on a besoin de remplacer des caractères spéciaux (genre le /).
Plutôt que de se casser la tête avec des échappements :

Code BASH :

sed 's/\/var\/log\/httpd/\/srv\/www\/logs/' /tmp/sed.txt

On pourra changer le séparateur, c'est beaucoup plus lisible. J'ai l'habitude d'utiliser l'arobase mais prenez celui que vous voulez :

Code BASH :

sed 's@/var/log/httpd@/srv/www/logs@' /tmp/sed.txt

Un truc que j'utilise aussi, c'est pour écrire en haut d'un fichier. Pour ça on peut le faire ainsi :

Code BASH :

sed '1s/^/Du texte en haut\n/' /tmp/sed.txt
