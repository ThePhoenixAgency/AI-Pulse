How to bring back macOS Launchpad with this neat command-line tool

Source: AppleInsider | Date: 2/11/2026 5:10:27 AM | Lang: EN | Original Article

Apple power users on macOS Tahoe are deliberately undoing parts of Apple's latest interface overhaul in order to remove the Liquid Glass design.

There's an unofficial workaround that brings back the classic Launchpad app grid and reduces Liquid Glass's heavy translucency. The tool allows users to enjoy macOS Tahoe's performance and feature updates without fully embracing its new visual style.

Liquid Glass looks polished in screenshots and marketing videos, with layered blur and reflections giving a sense of depth and motion. In daily use, those effects can feel intrusive, particularly on large displays or detailed wallpapers.

Menus and controls lose contrast, and text has to compete with background content. The interface demands more visual attention than many productivity-focused users are willing to give.

The familiar full-screen grid of apps in Launchpad, manually organized into pages and folders, is replaced by Spotlight's expanded search experience and an iOS-style App Library that automatically categorizes applications. Spotlight is more capable now, and some users find it faster than browsing.

However, it also shifts focus away from visual memory and hands-on organization, which have been central to many longstanding Mac workflows.

Apple doesn't provide a system setting to reverse changes like bringing back Launchpad or disabling Liquid Glass. Because of this, users who are comfortable with technology are turning to community-developed tools.

These tools selectively roll back specific design decisions without permanently altering macOS.

Why Apple moved away from Launchpad

Apple has been integrating app discovery into Spotlight across macOS, iOS, and iPadOS for years. Spotlight has evolved from a simple file search tool into a systemwide launcher and content index, absorbing features that once lived in separate interfaces.

On macOS, recent updates emphasize Spotlight actions, inline results, and contextual suggestions, reducing reliance on dedicated launch views like Launchpad. This change mirrors Apple's approach on iOS and iPadOS, where App Library replaced manual home screen organization as the default app-management model.

The approach prioritizes search-first navigation as app libraries grow and reduces the need for users to manage visual layouts. For longtime Mac users, the shift trades discoverability and visual memory for speed, offering fewer options for those who prefer browsing over typing.

What the Undye workaround actually does

The solution most users have converged on is an open-source tool called Undye, available on GitHub at Launchpad Revived. Undye automates the entire process and makes each step explicit.

Under the hood, Undye replaces macOS Tahoe's Dock with an older Apple-signed Dock binary from an early macOS 26 beta, back when Launchpad was still part of the system. A binary is a compiled system executable, and in this case it's one produced and signed by Apple itself, not modified or patched by a third party.

Undye also reinstalls the Launchpad app, which Apple removed from later macOS Tahoe builds.

Because these components are authentic and unaltered, macOS recognizes them as legitimate system software. It avoids code injection, memory patching, or hidden feature flags.

Launchpad is restored and behaves as it did in earlier macOS versions, with reduced translucency across menus, Control Center, and other system panels.

Launchpad as it appeared before macOS Tahoe

This approach isn't supported by Apple, and a future macOS update could break compatibility. However, because Undye relies on Apple-signed binaries rather than exploits or hacks, it has remained functional through several macOS Tahoe updates.

If Apple tightens validation around Dock components, users would need to remove the workaround before updating.

Who should and should not attempt this

This workaround is intended for personal Macs that users fully own and administer themselves. It's best suited for people comfortable working in macOS Recovery, entering Terminal commands, and maintaining reliable system backups.

It should not be used on work-issued Macs, managed systems, or devices governed by mobile device management policies. Anyone uncomfortable temporarily disabling System Integrity Protection or restoring from a backup should avoid the process altogether.

Because Undye modifies system files, Apple doesn't support the configuration. Apple Support may ask users to revert the changes before troubleshooting unrelated hardware or software issues.

Step one: download Undye

Open a web browser and navigate to the Undye GitHub repository at GitHub. Click the Code button and download the project as a ZIP file.

ZIP preserves file permissions and ensures you receive the complete, unmodified project. After downloading, extract the archive and move the resulting folder to a convenient location, such as a newly created folder inside Downloads.

The extracted folder contains a single script along with the Apple-signed system components Undye installs during the process.

Step two: disable System Integrity Protection

System Integrity Protection prevents modification of critical macOS components and must be disabled temporarily for the workaround to function.

  1. Restart the Mac.
  2. Enter macOS Recovery.
  3. On Intel Macs, immediately press and hold Command + R until the Apple logo or spinning globe appears.
  4. On Apple Silicon Macs, shut down the Mac, then press and hold the power button until "Loading startup options" appears, select Options, and click Continue.
  5. Once the macOS Utilities screen appears, open Terminal from the Utilities menu.

In Terminal, enter:

csrutil disable

After receiving confirmation, restart the Mac normally.

Step three: run the Undye script

Open Terminal in macOS and navigate to the extracted Undye folder. Carefully copy and paste the command below, making sure it's entered exactly as written.

sudo ./launchpad

Enter your administrator password when prompted. The script replaces the Dock, reinstalls Launchpad, and restarts the necessary system services automatically.

During execution, Undye prints code-signing information so users can confirm that the installed components are signed by Apple.

Confirming Launchpad and reduced translucency

Once the script finishes, Launchpad should appear in the Dock or respond to its usual keyboard shortcut. Opening it reveals the same paginated grid of apps with folder support that longtime Mac users expect.

System UI elements such as Control Center and menus should appear more opaque and readable than before. Additional opacity can be enabled separately using the Reduce Transparency option in System Settings under Accessibility, which is independent of Undye.

Spotlight and App Library remain available alongside restored Launchpad

Dynamic icons such as Calendar appear static within Launchpad, as they did in older macOS versions. Minor visual quirks may appear in some Finder views or system panels, but these are cosmetic and don't affect core functionality.

Re-enabling System Integrity Protection

After confirming everything works as expected, System Integrity Protection should be re-enabled.

Restart into macOS Recovery, open Terminal, and run:

csrutil enable

Restart the Mac normally. The restored Dock and Launchpad continue working with System Integrity Protection enabled.

The Undye script is fully reversible. Running the same script again with System Integrity Protection disabled restores Apple's original Dock and removes Launchpad.

After reverting, users can re-enable System Integrity Protection and return to macOS Tahoe's default Liquid Glass interface and Spotlight-based app launching. Apple may change or block this behavior in future releases, so users should reassess compatibility before major system updates.
