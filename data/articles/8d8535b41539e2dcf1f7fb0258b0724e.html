Agent Workflow

How agents use Corral

Six steps from "add auth" to shipping. No human touches auth code.

1

Agent reads llm.txt You paste a full URL like https://corral.llamafarm.dev/llm.txt into your agent context, or add that exact URL to AGENTS.md / .cursorrules. The agent now knows every command, every endpoint, every option.

2

Agent runs npm create corral Scaffolds corral.yaml, route handlers, components, and CORRAL.md into your project. Non-interactive. Idempotent.

3

Agent edits corral.yaml for your plans Free tier, Pro at $29/mo, Enterprise — whatever you asked for. Auth providers, feature flags, usage meters. All in one file.

4

Agent runs corral add feature, corral add provider Incremental commands for each capability. Idempotent — safe to re-run. --json output on every command so the agent can branch on results.

5

Agent runs corral validate — all green Checks config, env vars, database, Stripe sync. Exit 0 means deploy-ready. If something's wrong, the JSON error tells the agent exactly how to fix it.

6

You have auth + payments. Ship it. No human touched auth code. No auth docs were read. The agent handled everything.

Agent-Native Design

Every detail built for agents

An agent can add full auth + billing to any app in 9 commands. No screen-scraping. No ambiguity. Just structured I/O.

/api/llms.txt

Auto-generated endpoint on every Corral app. Describes the project's auth config, active plans, feature gates, and API surface in a format LLMs parse cleanly. Follows the llms.txt spec. Also served at llms.txt.

CORRAL.md

Generated at init time and updated on every corral add. A project-specific guide: what's configured, what routes exist, what gates are active, how to test auth locally. Designed for agent system prompts.

corral add <thing>

Incremental commands that do one thing well. corral add feature ai_analysis --plans pro --gate blur. Idempotent. Safe to run twice. Agent-friendly JSON output.

--json everywhere

Every CLI command accepts --json and exits with structured output + a numeric exit code. Parse it, branch on it, pass it to the next command. No screen-scraping.

The 9-command workflow — from zero to auth + billing:

agent workflow

corral analyze --json # 1. understand the project corral init --yes --json # 2. scaffold everything corral add provider google # 3. add OAuth corral add plan pro --price 29 # 4. add paid plan corral add feature ai_features --plans pro # 5. gate a feature corral add meter api_calls --limit 10000 # 6. add usage meter corral stripe push --json # 7. sync to Stripe corral validate --json # 8. verify everything corral seed --env test --json # 9. seed test users
