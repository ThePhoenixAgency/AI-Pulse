I Built a Free Keyword Tool Because Every Paid Alternative Is $99/mo — Here's What I Learned

Source: Dev.to Open Source | Date: 2/18/2026 4:46:23 AM | Lien | Lang: EN

A few weeks ago, I wrote a frustrated post called Every Google Ads Keyword Tool Is Full of Ads. It resonated — 42 views and counting, which made it my most-read article. Clearly, I wasn't the only one annoyed. But complaining only gets you so far. So I did what any developer would do: I built my own. If you've ever tried to do keyword research without paying $99/mo for Ahrefs, SEMrush, or similar tools, you know the pain: Free tools bombard you with upsells Google Keyword Planner requires an active ad campaign Most "free" alternatives give you 3 searches before demanding a credit card The data you actually need is always behind a paywall I'm a solo developer. I run a blog. I don't need enterprise-grade SEO analytics. I just need to know: what are people actually searching for? I spent a couple of weekends building a completely free, open-source keyword research tool. No login required. No usage limits. No "upgrade to pro" popups. Try it here Here's what it does: Keyword suggestions from Google Autocomplete API Related searches and question-based keywords Search volume estimates where available Export to CSV for further analysis Works entirely in the browser — no backend needed I kept it deliberately simple: Vanilla JavaScript — no React, no Vue, no framework overhead GitHub Pages — free hosting, zero maintenance Google Autocomplete API — the same suggestions Google shows when you type Pure CSS — responsive, clean, no Tailwind dependency Why no framework? Because this is a tool, not a SaaS product. It needs to load fast, work reliably, and cost me nothing to host.
