madari

Madari handles the installation of MCP servers and their registration with Claude Desktop with a single command. As the MCP ecosystem grows, so does the need to manage the mesh of AI clients and mcp servers. Madari gives you one local control point to install, register, sync, and diagnose your mcp setup.

Need for 

madari
grows as AI agents start picking their own tools!

Commands

install

install
is the fastest way to setup a working MCP server:

madari install stewreads-mcp
  • runs 
    uv tool install
  • resolves the binary path
  • registers the server in Madari's registry
  • syncs it to Claude Desktop

Everything else

Lifecycle - 

add
, 
remove
, 
enable
, 
disable
, 
list

Register servers manually, toggle them on or off, and inspect what's in your registry.

Sync - 

sync claude-desktop [--dry-run]

Push registry state to Claude Desktop's config. Madari only touches entries it registered and backs up your config before every write.

Diagnostics - 

doctor
, 
status

Validate command paths, required environment variables, and client config health.

Snapshots - 

export
, 
import

Back up your full registry or move it across machines.

Installation

Platform support:

  • macOS, Linux, and Windows.
  • Claude Desktop is the current sync target. More client support coming soon!
Examples 
# Install a standard MCP server package in one shot:
madari install mcp-server-git # Install a package with dots in its name by providing a valid Madari server name (known issue tracked in #9):
madari install awslabs.core-mcp-server \ --name awslabs-core-mcp-server \ --skip-install # Add an npx-based filesystem server with explicit allowed directories:
madari add filesystem \ --command npx \ --client claude-desktop \ --arg -y \ --arg @modelcontextprotocol/server-filesystem \ --arg ~/Downloads \ --arg ~/Documents # Sync to Claude Desktop, then run diagnostics:
madari sync claude-desktop

Reload Claude Desktop MCP configuration (or restart Claude Desktop) to apply changes. All 3 MCP servers should become available in Claude Desktop.

# inspect current health of all MCP servers:
madari doctor
madari status

Removing MCP servers:

madari remove awslabs-core-mcp-server
madari remove filesystem
madari sync claude-desktop

Reload Claude Desktop MCP configuration (or restart Claude Desktop) to apply changes. Run the diagnostic commands again, these 2 MCP servers will be removed from Claude Desktop.
