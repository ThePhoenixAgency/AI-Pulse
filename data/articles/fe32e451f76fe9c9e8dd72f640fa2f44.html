How Code Scalpel actually works

No magic. No heuristics. Code Scalpel uses real computer science — the same techniques compilers and formal verifiers use — and gives them directly to your AI agent.

Real Syntax Trees

Code gets parsed into an Abstract Syntax Tree — the same structure a compiler sees. Your agent reads real code structure, not guessed patterns from text.

Dependency Graphs

Every function call, import, and reference is mapped into a call graph. Your agent sees exactly what connects to what — across every file in your project.

Taint Analysis

User input is traced through every code path to where it's used — SQL queries, file writes, system calls. If data reaches a dangerous spot unsanitized, your agent catches it.

Formal Verification

The Z3 theorem prover explores every possible execution path through a function. Your agent can mathematically prove a refactor won't change behavior.
