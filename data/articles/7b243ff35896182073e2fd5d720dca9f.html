Dragino x Raspberry Pi : sélection MONRASPBERRY et guides d’intégration IoT (LoRaWAN & industriels)

Pourquoi Dragino + Raspberry Pi ?

  • Compatibilité IoT éprouvée : capteurs LoRaWAN prêts pour EU868, modules RS485-LoRa pour l’industriel, HAT LoRa pour P2P/tests.
  • Faciles à déployer : profils LoRaWAN connus, payloads documentés, alimentation sobre.
  • Parfaits avec un Pi : collecte locale (InfluxDB/Grafana/Node-RED), ou via une passerelle ChirpStack sur Raspberry Pi 5 (recommandé dans la boutique).

Les références disponibles

  • LHT65N — Capteur T°/Humidité IP65 (extérieur/indus).
  • LHT52 — Capteur T°/Humidité intérieur (ultra basse conso).
  • AQS01-L — Capteur qualité de l’air intérieur (CO₂, T°, HR, parfois pression).
  • LWL02 — Détecteur de fuite d’eau (compact, autonome).
  • LWL04 — Détecteur de fuite (format “rampe” / ligne), usage continu.
  • LDS02 — Ouverture/fermeture de porte (contact magnétique).
  • SE01 — Humidité du sol / température / conductivité (EC) (Jardins, serres, cuves).

Choisir le bon capteur Dragino selon ton projet

1) Bâtiment / IT / tertiaire (indoor)

  • Confort & IAQ : AQS01-L pour CO₂/TVOC + T°/HR → salles de classe, open spaces, salles de réunion.
  • Confort thermique : LHT52 (intérieur) pour T°/HR rooms, vitrines, chambres froides légères.
  • Fuites techniques : LWL02 (spot) ou LWL04 (ligne) → faux-planchers, racks, locaux techniques.
  • Contrôle d’accès léger : LDS02 sur portes/armoires.

2) Extérieur / agro / indus léger

  • T°/HR robuste : LHT65N (IP65) → zones semi-exposées, entrepôts, boxes.
  • Sol & irrigation : SE01 (humidité/EC) → bacs, serres, parcelles, citernes.
  • Détection fuites étendue : LWL04 le long des tuyauteries.

Architecture type avec Raspberry Pi (LoRaWAN)

Passerelle conseillée : Raspberry Pi 5 (4GB) + concentrateur LoRa SX1302 (HAT/LR1302), antenne EU868, alim USB-C 45 W, microSD 32 Go.
Stack logicielle : ChirpStack Gateway OS (ou Docker) → Gateway Bridge, Network Server, Application Server.
Flux : Capteur Dragino → radio LoRa → Passerelle Pi → MQTT → Décodage (codecs) → Tableau de bord (Node-RED / Grafana) ou envoi API.

Bonnes pratiques
• Déclare tes capteurs en OTAA, EU868.
• Défini un Device Profile par famille (T°/HR, IAQ, fuites, sol).
• Active un codec (Cayenne LPP ou spécifique Dragino) pour obtenir des champs lisibles (temperature, humidity, co2, leak, open, soil_moisture, ec…).

Pas-à-pas (exemples concrets)

A) Confort & qualité de l’air (AQS01-L + LHT52)

  1. Passerelle Pi prête (ChirpStack OK).
  2. Device Profile IAQ (classe A, RX2 EU par défaut, DR0).
  3. Device : saisis DevEUI / JoinEUI / AppKey du AQS01-L.
  4. Codec : IAQ/CO₂.
  5. Dashboard :
    • Panneau CO₂ (ppm) + T° (°C) + HR (%).
    • Seuils (ex. 1000 ppm) → alerte.
  6. Ajoute LHT52 pour comparer CO₂ vs confort (T°/HR) par salle.

B) Anti-fuite (LWL02 spot & LWL04 en ligne)

  1. Device Profile “leak” (uplink événementiel + heartbeat).
  2. LWL02 : positions ciblées (sous clim, lave-vaisselle, nourrices).
  3. LWL04 : pose le long des rampes/tuyaux (zones longues).
  4. Règles : si leak == true → alerte immédiate + journalisation + éventuellement coupure électrovanne (via relais sur une autre brique).

C) Agro / irrigation (SE01 + LHT65N)

  1. SE01 : renseigne les paramètres de remontée (ex. toutes les 15 min).
  2. LHT65N : ambiance extérieure robuste (IP65) pour corréler T°/HR avec humidité/EC du sol.
  3. Scénario :
    • Si soil_moisture < seuil ET ETP forte (forte T°/faible HR) → notification “Irrigation conseillée”.
    • Historise EC pour surveiller la salinité et adapter l’arrosage.

D) Supervision d’accès léger (LDS02)

  1. Pose LDS02 sur portes/racks.
  2. Dash : nombre d’ouvertures / plages horaires.
  3. Alerte si ouverture en dehors des heures (downlink possible pour ajuster paramètres).

Décodage : formats & routage (exemples)

MQTT sortant (ChirpStack)

  • Topic type : application/<appID>/device/<devEUI>/event/up
  • Payload JSON (ex. T°/HR) :
{ "deviceInfo": {"devEui": "A84041...", "name": "lht52_salle_a"}, "rxInfo": [{"rssi": -92, "snr": 6.5}], "object": {"temperature": 22.6, "humidity": 47.8, "battery": 3.01}
}
  • IAQ (ex. AQS01-L) :
{"co2": 812, "temperature": 23.1, "humidity": 40.3, "pressure": 1007}
  • Fuite (LWL02/LWL04) :
{"leak": true, "counter": 3}
  • Sol (SE01) :
{"soil_moisture": 28.5, "soil_temp": 18.4, "ec": 1.12}

Astuce : route vers Node-RED pour les règles (seuils/alertes), InfluxDB pour l’historique, Grafana pour les dashboards.

Autonomie & réglages typiques

  • LHT52 / LHT65N : intervalle 5–15 min (bâtiment) ou 15–30 min (extérieur) → multi-années.
  • AQS01-L : intervalle 5–10 min en intérieur (équilibre granularité/autonomie).
  • LWL02/LWL04 : événementiel + heartbeat (ex. 60–120 min).
  • SE01 : 10–30 min selon dynamique du sol (plus lent que l’air).

(Toujours ajuster en fonction de la batterie et du duty-cycle EU868.)

Bonnes pratiques d’installation

  • Antenne de la passerelle : en hauteur, câble court (type LMR-400), gain 3–8 dBi.
  • Capteurs indoor : éviter zones de courants d’air (CO₂, T°/HR).
  • Capteurs fuites : poser au point bas ; pour LWL04, longer la zone sensible.
  • Capteurs sol : bien tasser autour de la sonde, calibrer si nécessaire (EC).
  • Sécurité : MQTT en TLS, VLAN dédié IoT, mises à jour régulières du Pi.
