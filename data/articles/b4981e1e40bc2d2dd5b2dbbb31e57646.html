v1.2.0 - Improved DB Compatibility, Extended JSON Type With Collections & Docker Reliability

This release focuses on improved database migration compatibility, Docker Compose reliability, and new JSON type support—along with internal refactoring and better overall developer experience.

Improvements

  • Internal project refactoring to improve maintainability, structure, and future extensibility.
  • Better migration compatibility across MySQL, MSSQL, and PostgreSQL (more robust Flyway script generation).
  • Improved support for newer MySQL versions (fixed compatibility issues with MySQL versions newer than 8.4).

Fixes

  • Fixed Flyway scripts when using reserved SQL keywords (reserved keywords are now supported).
  • Fixed naming of unique constraints to ensure consistent and correct constraint generation.
  • Fixed Docker Compose exposed vs internal ports configuration.
  • Updated 
    .openapi-generator-ignore
    to prevent regenerating/overwriting 
    pom.xml
     and README files (
    README.md
    , 
    Readme.md
    , 
    readme.md
    ).

New Features

  • Added support for a new JSON type: 
    • JSON<List<Type>>
    • JSON<Set<Type>>

Docker / Dev Experience

  • Added healthchecks to Docker Compose so the Spring Boot application starts only after the database(s) are ready.
  • Added a project banner (includes version and source file/output path information).
