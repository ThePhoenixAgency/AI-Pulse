I Built 5 Developer Tools That Run Entirely in Your Browser

Source: Dev.to Open Source | Date: 2/17/2026 5:16:08 PM | Lien | Lang: EN

I got tired of signing up for yet another SaaS just to encode an HTML entity or generate a QR code. So I built my own — all browser-based, zero backend, zero tracking.

Here are 5 tools I actually use daily. They're free, open source, and work offline after first load.

1. HTML Entity Encoder

Problem: You're writing a blog post and need to escape <div> for display. You Google "html entity encoder", land on a site with 14 ads, and question your life choices.

Solution: Paste text → get encoded output. That's it.

Try it

Works with all named entities (&amp;, &lt;, &gt;) and numeric codes. Bidirectional — encode or decode.

2. Markdown Preview

Problem: You're editing a README and want to see how it renders without pushing to GitHub 47 times.

Solution: Live preview as you type. Split pane. GitHub-flavored markdown support.

Try it

Supports tables, code blocks with syntax highlighting, task lists, and emoji shortcodes.

3. QR Code Generator

Problem: Your PM asks for a QR code linking to the staging URL. "Can you make it... now?"

Solution: Type/paste any URL or text → instant QR code → download as PNG.

Try it

Customizable size. Works with URLs, text, WiFi configs, vCards — anything that fits in a QR code.

4. Color Picker & Palette Generator

Problem: "What's the hex for that blue?" opens DevTools, inspects, copies, pastes into another tool to get HSL...

Solution: Pick a color → get HEX, RGB, HSL instantly. Generate complementary/analogous palettes with one click.

Try it

Copy any format to clipboard. Export palettes as CSS variables.

5. Word & Character Counter

Problem: "Your meta description must be under 160 characters." opens Google Docs, pastes, checks status bar...

Solution: Paste text → word count, character count (with/without spaces), sentence count, reading time.

Try it

Perfect for SEO meta descriptions, tweet drafts, or any length-constrained writing.

Why browser-based tools?

I have a simple rule: if it doesn't need a server, don't use a server.

  • No install — bookmark it, done
  • No signup — no email, no password, no "verify your account"
  • No data collection — everything stays in your browser
  • Works offline — after first load, no internet needed
  • Open source — fork it, improve it, make it yours
